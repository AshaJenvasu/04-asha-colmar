# Project Reflection

For this assessment, my core approach was to treat every element as a **"box within a box."**  
By structuring the entire layout as a primary Flexbox containing multiple nested Flexboxes, I was able to control the arrangement of components freely—whether in rows or columns—while maintaining a clean and scalable structure.

---

## Approach & Decision Making

Although Tailwind CSS was recommended, I chose to focus on **pure CSS** for this project.  
Since I am still developing my understanding of Tailwind’s utility-based approach, I believed it was more important to first build a strong foundation in CSS.

By doing so, I was able to better understand layout behavior, spacing, and responsiveness at a deeper level, which will also support my future use of frameworks like Tailwind and React.

---

## Development Process

I started from the top of the layout and worked my way down systematically.  
To improve efficiency and accuracy, I kept the following open at all times:

- Reference design
- VS Code
- Live preview

Using Flexbox as the main layout system, I structured each section into smaller components and controlled spacing and alignment using properties such as `justify-content` and width percentages.

As the layout became more complex, I applied nested Flexbox structures to divide sections into smaller content blocks, combining both row and column directions depending on the design requirements. This approach allowed me to manage multiple layers of content while keeping the code organized and reusable.

---

## Responsiveness Challenges

The most challenging part of this project was implementing **responsive design**.

I used the following breakpoint as a standard: @media (max-width: 768px)

In some cases, I also worked with separate structures for desktop and mobile layouts to better match the provided assets. While this increased the workload, it helped me achieve more precise visual results.

---

## Key Learnings

Through this project, I improved my ability to:

- Structure layouts using nested Flexbox
- Manage complex sections with multiple layers of content
- Design with responsiveness in mind
- Write more organized and readable CSS

I also experimented with a **mobile-first approach** in later parts of the project, which helped simplify responsive adjustments and made the layout more predictable across devices.

---

## Reflection

I think this project was very fun. Seeing the design come to life exactly as intended gave me a strong sense of satisfaction.

Using CSS throughout this project significantly improved my understanding of layout and styling. I especially liked how the code became more organized and easier to read, which made it much more enjoyable to revisit and fix issues.

Overall, this experience helped me feel more confident in my CSS skills and motivated me to continue improving further.
